Contexto

El problema que existía antes

Antes de MCP, cada integración entre un modelo de IA y una herramienta externa requería código personalizado.

1

Conectar Claude a Notion

Conector propio, mantenimiento continuo.

2

Conectar Claude a GitHub

Otro conector distinto, otra API que aprender.

3

Conectar Claude a una base de datos

Un tercer desarrollo, con su propio código.

El resultado

Un problema N × M. N modelos de IA × M herramientas = caos de integraciones. Como tener un enchufe diferente para cada país del mundo.

Antes vs Después

Antes: Caos punto a punto

Cada modelo necesitaba su propio conector para cada herramienta. Imposible de escalar. Costoso de mantener.

Ahora: Hub central con MCP

Un solo protocolo. Cualquier cliente MCP se conecta a cualquier servidor MCP. Escribís la integración una vez y funciona con todos.

Definición

¿Qué es MCP?

MCP (Model Context Protocol) es un estándar abierto que define una forma universal para que los modelos de IA se conecten a herramientas, datos y servicios externos.

Creado por

Anthropic

Lanzado en

Noviembre 2024

Basado en

JSON-RPC 2.0

Inspirado en

LSP (editores de código)

USB-C

MCP es el USB-C de la IA

Un conector universal que funciona con cualquier dispositivo. Así como USB-C eliminó la necesidad de cables propietarios, MCP elimina la necesidad de integraciones punto a punto entre cada IA y cada herramienta.

Arquitectura

Cómo funciona MCP

MCP define dos roles en su arquitectura:

MCP Server

Expone herramientas y datos. Es el que sabe hacer algo: leer correos, consultar una base de datos, crear un issue en GitHub.

MCP Client

Se conecta al servidor y usa sus capacidades. Claude, ChatGPT, Gemini y otros agentes actúan como clientes MCP.

Flujo básico en 5 pasos

1

El usuario le pide algo a Claude

"Revisá mis correos de hoy y resumime los urgentes."

2

Claude detecta qué herramienta necesita

Identifica que necesita el servidor MCP de Gmail.

3

Claude llama al MCP Server correspondiente

Envía una solicitud estándar por JSON-RPC.

4

El servidor ejecuta y devuelve el resultado

Gmail responde con la lista de correos del día.

5

Claude usa el resultado para responder

Procesa los correos y presenta el resumen al usuario.

Historia

Historia y adopción

Noviembre 2024

Anthropic lanza MCP

Se publica como estándar abierto bajo la especificación Model Context Protocol, basado en JSON-RPC 2.0.

2025

El ecosistema explota

GitHub, Postgres, Notion, Stripe, Figma y cientos de herramientas más adoptan MCP. Cursor, VS Code y Windsurf lo integran nativamente. Más de 10.000 servidores MCP públicos activos.

Noviembre 2025

Donación a la Agentic AI Foundation

Anthropic dona MCP a la fundación respaldada por la Linux Foundation, co-fundada con Block y OpenAI. ChatGPT, Gemini y Microsoft Copilot anuncian su adopción.

Diciembre 2025

Estándar de facto de la industria

MCP se consolida como el estándar universal para la conexión entre agentes de IA y el mundo exterior.

10K+

Servidores MCP activos

Comunidad global de desarrolladores creando y publicando servidores MCP para todo tipo de herramientas y servicios.

Gobernanza

MCP es gobernado por la Agentic AI Foundation, parte de la Linux Foundation, con Anthropic, Block y OpenAI como co-fundadores.

Comparativa

MCP vs. integraciones tradicionales

Factor Integración tradicional Con MCP
Tiempo de setup Días o semanas. Cada integración requiere desarrollo a medida. Horas. El servidor MCP se escribe una vez y funciona con cualquier cliente.
Mantenimiento Alto. Hay que mantener un conector por cada modelo de IA. Una sola vez. El servidor MCP se mantiene independiente del cliente.
Compatibilidad Solo funciona con el modelo para el que se desarrolló. Cualquier cliente MCP (Claude, ChatGPT, Gemini, Copilot).
Estándar Propietario y cerrado. Atado a un proveedor. Abierto. Gobernado por la Linux Foundation.

El cambio fundamental

Con MCP, la integración se convierte en un activo reutilizable. La escribís una vez y funciona con Claude, ChatGPT, Gemini y cualquier agente que implemente el protocolo.

Arquitectura

El stack moderno del agente de IA

MCP es una pieza de un ecosistema que se consolidó en 2025. Cada capa resuelve un problema distinto.

Capa 1

Agente de IA

Claude, GPT, Gemini, Copilot

Capa 2

MCP

Cómo se conecta a datos y herramientas

Capa 3

Agent Skills

Cómo ejecuta tareas con criterio especializado

Capa 4

Herramientas y fuentes de datos

Qué puede usar: APIs, bases de datos, archivos

La distinción clave

MCP resuelve el cómo conectar. Las Skills resuelven el cómo hacer. Juntos forman el dual-stack de la era moderna de agentes de IA.

Analogía

MCP es el sistema operativo que le da acceso al hardware. Las Skills son las aplicaciones que saben qué hacer con ese acceso.

Negocios

Relevancia para el mundo empresarial

Si trabajás con ERP, auditoría, contabilidad o sistemas empresariales, MCP cambia las reglas del juego:

1

Leer tu ERP sin integraciones costosas

Un agente con MCP puede consultar SAP, Dynamics u Oracle directamente, sin necesidad de desarrollar conectores a medida.

2

Cruzar datos entre sistemas

El agente puede consultar el ERP, el CRM y el correo simultáneamente y cruzar información que hoy vive en silos.

3

Automatizaciones en días, no meses

Lo que antes requería un proyecto de integración de semanas, con MCP se configura en horas.

4

Sin vendor lock-in

Al ser un estándar abierto gobernado por la Linux Foundation, no quedás atado a un solo proveedor de IA.

Caso concreto

A

Un agente consulta el ERP para obtener el listado de facturas pendientes de conciliación.

B

Cruza esa información con los extractos bancarios obtenidos vía otro servidor MCP.

C

Genera la conciliación automática y marca las discrepancias para revisión humana.

D

Envía el reporte por correo y agenda una reunión de revisión en el calendario del equipo.

De punta a punta, sin intervención manual.
