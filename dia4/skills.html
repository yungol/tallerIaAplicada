Definición

¿Qué es una skill?

Una skill es un archivo de instrucciones en Markdown que le enseña a un agente de IA cómo realizar una tarea específica. Es el equivalente digital de un manual de oficio: contiene el conocimiento procedural, las reglas, las referencias y los patrones que el agente necesita para ejecutar con criterio.

Analogía

Si el modelo de lenguaje es un profesional recién graduado con conocimiento teórico enciclopédico, una skill es el manual del puesto que le explica cómo aplicar ese conocimiento en un contexto real, siguiendo las convenciones y criterios de un equipo concreto.

Anatomía de una skill

1

Frontmatter YAML

Nombre, descripción, triggers que activan la skill.

2

Instrucciones procedurales

Workflow paso a paso, reglas de decisión, criterios de calidad.

3

Referencias y contexto

Archivos adjuntos, ejemplos, documentación de dominio.

4

Directorio autocontenido

Todo lo que la skill necesita vive en su propia carpeta.

Propósito

¿Para qué sirve una skill?

Especialización

Convierte un modelo generalista en un experto para un dominio concreto. La skill encapsula el conocimiento tácito que normalmente solo existe en la cabeza del equipo.

Consistencia

Garantiza que el agente siempre siga las mismas convenciones, patrones y criterios de calidad. Cada ejecución produce resultados predecibles y alineados con el estándar del equipo.

Reutilización

Una skill bien escrita se comparte entre proyectos, equipos y herramientas. Es el equivalente a un paquete de npm pero para conocimiento operativo de agentes de IA.

El cambio de paradigma

Antes le pedíamos al modelo que improvisara cada vez. Con las skills, le entregamos un estándar escrito que puede seguir, refinar y mejorar iterativamente. Pasamos de prompts artesanales a instrucción ingenieril.

Historia

¿Quién creó las skills?

Noviembre 2024

Model Context Protocol (MCP)

Anthropic lanza MCP como estándar abierto. Es la infraestructura sobre la que luego se construyen las skills: estandariza cómo Claude se conecta a herramientas y fuentes de datos externas.

Octubre 2025

Skills (primera versión)

Anthropic introduce Skills como patrón formal para extender las capacidades de Claude con conocimiento especializado y flujos de trabajo. Cada skill es una carpeta con un archivo SKILL.md que Claude descubre y carga dinámicamente según la tarea.

Diciembre 2025

Agent Skills como estándar abierto

Anthropic lanza oficialmente Agent Skills como estándar abierto y lo dona a la Agentic AI Foundation (respaldada por la Linux Foundation). Junto con MCP forma el dual-stack de la era moderna de IA: MCP define cómo los agentes se conectan a datos, Agent Skills define cómo ejecutan tareas.

Febrero 2026

Agent Teams

Se lanzan oficialmente los Agent Teams: instancias independientes de Claude Code que se comunican entre sí, llevando el ecosistema de skills y agentes un paso más allá.

Creador

Anthropic

La empresa detrás de Claude creó las skills como patrón formal para extender agentes de IA con conocimiento especializado. El estándar fue donado a la Agentic AI Foundation, respaldada por la Linux Foundation.

Dual-stack

MCP + Agent Skills es el dual-stack de la era moderna de IA: MCP define cómo los agentes se conectan a datos y herramientas; Agent Skills define cómo ejecutan tareas con criterio especializado.

En resumen

Key takeaway

Una skill es un manual de oficio digital que transforma un agente de IA generalista en un especialista con criterio. Es el puente entre el conocimiento teórico del modelo y la práctica concreta de un dominio.

Para recordar

  • 1. Son modulares, autocontenidas y reutilizables.
  • 2. Usan Markdown + YAML: simples de escribir, poderosas en ejecución.
  • 3. Nacieron en Anthropic pero son un formato abierto e interoperable.
  • 4. El ecosistema crece en skills.sh.

∞

Posibilidades

Cada dominio, cada oficio, cada especialidad puede codificarse como una skill. El límite lo pone la comunidad.
